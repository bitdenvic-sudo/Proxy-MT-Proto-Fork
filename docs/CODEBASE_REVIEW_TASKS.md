## ✅ 1) Задача: привести документацию и комментарии к фактическому поведению `get_public_ip`

### Наблюдения (решено в v5.0)
- В `README.md` и `USERGUIDE.md` заявлена поддержка IPv6 и перечислены конкретные источники IP, которых нет в коде (например, `ifconfig.co`, `ipinfo.io`).
- В `src/utils.sh` функция `get_public_ip()` использует только IPv4-валидацию через `validate_ip()` и фактически не может вернуть IPv6.
- Из-за этого текущее описание возможностей расходится с реальной реализацией.

### Решение (v5.0)
**Status:** ✅ **ВЫПОЛНЕНО**

1. **Добавлена поддержка IPv6**: Создана новая функция `get_public_ip_v6()` в `src/utils.sh` с:
   - 3 внешними источниками IPv6 (api6.ipify.org, ifconfig.me/ip6, icanhazip.com)
   - Локальным fallback через `hostname -I`
   - Логированием процесса определения IP

2. **Обновлена документация**:
   - `README.md`: уточнено, что `get_public_ip()` возвращает IPv4, добавлено описание `get_public_ip_v6()`
   - Исправлено количество источников: 4 для IPv4 + 3 для IPv6

3. **Обновлены комментарии в коде**:
   - Добавлен комментарий в `get_public_ip()`: "Note: This function returns IPv4 only. For IPv6 support, use get_public_ip_v6()."
   - Добавлено логирование через `LOG_DEBUG` для отладки

4. **Changelog**: Добавлена запись в `CHANGELOG.md` о поддержке IPv6 в v5.0

## ⏳ 2) Задача: улучшить тест `get_public_ip`, убрать нестабильность от внешней сети

### Наблюдения
- Текущий тест в `tests/test_utils.bats` вызывает реальные внешние сервисы.
- Поведение теста зависит от сети/доступности DNS и часто не проверяет детерминированный результат.

### Предлагаемая задача (issue)
**Title:** `tests(utils): сделать get_public_ip детерминированным через моки curl/hostname`

**Definition of Done:**
1. Переписать тест так, чтобы он не зависел от интернета:
   - мок `curl` для сценариев «несколько неудач, потом валидный ответ»;
   - мок `hostname -I` для проверки локального fallback.
2. Добавить минимум 3 сценария:
   - успех на первом сервисе;
   - успех на N-м сервисе после невалидных ответов;
   - все сервисы неуспешны, fallback через `hostname -I`.
3. Явно проверять и код возврата, и итоговый IP.
4. Оставить один отдельный smoke/integration тест для реальной сети как опциональный (по env-флагу), а не как обязательный unit.

**Priority:** Medium  
**Milestone:** v5.1
