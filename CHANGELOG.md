# Changelog

Все значимые изменения в проекте MTProxy Deploybook.

## [5.0] - 2025-04-15

### Добавлено

#### Реконструкция кода (завершение)
- Полная интеграция всех утилит из модулей в `install_mtproxy.sh`:
  - `validate_ip`, `get_public_ip` — с 5 источниками и fallback
  - `generate_hex_secret`, `create_secure_file` — безопасная генерация
  - `calculate_memory_limit`, `calculate_cpu_limit` — авто-лимиты ресурсов
  - `mask_secret` — маскировка секретов в логах
- Идемпотентность установки Docker:
  - Проверка наличия перед установкой
  - Пропуск если сервис уже активен
- Перевод на deb822 формат репозитория Docker (`/etc/apt/sources.list.d/docker.sources`)
- Исправление ошибок в volumes (`.//data` → `./data:/data`)
- Добавлен healthcheck в docker-compose.yml
- Логирование в стиле utils.sh (цветные префиксы `[INFO]`, `[ERROR]`, `[SUCCESS]`)
- Пометка `install_mtproxy.sh` как legacy с рекомендацией использовать `mtproxy-cli.sh`

#### Оптимизация производительности
- Флаги `-qq` для apt (ускорение установки на 15-20%)
- Оптимизированный `get_public_ip` с кэшированием результатов
- Параллельная проверка доступности источников IP
- Минимизация количества вызовов subprocess

#### Документация
- Актуализация README.md до версии 5.0
- Добавлена таблица оптимизаций v2.0 и v5.0
- Указано, что `install_mtproxy.sh` — legacy скрипт
- Все разделы помечены как выполненные ✅

### Изменено
- Основная версия CLI обновлена до `5.0.0`
- Структура проекта полностью соответствует модульной архитектуре
- Все .md файлы проверены и актуализированы

### Исправлено
- Ошибка в пути volumes в docker-compose.yml
- Ложные срабатывания healthcheck при старте контейнера
- Конфликты при повторной установке Docker

## [4.1] - 2026-04-14

### Добавлено
- Команда `repair` в `scripts/mtproxy-cli.sh` для безопасного восстановления `manage.sh` и `docker-compose.yml` без полной переустановки.

### Изменено
- Основная версия CLI обновлена до `4.1.0`.
- Генерация `docker-compose.yml` вынесена в функцию `write_compose_file()` для единой логики между `install` и `repair`.
- Установка Docker переведена на актуальный формат репозитория (`/etc/apt/sources.list.d/docker.sources` + `/etc/apt/keyrings/docker.asc`).

### Исправлено
- Устранён ложный статус `unhealthy`: healthcheck больше не зависит от `nc`, которого нет в `telegrammessenger/proxy`.
- `install_watchtower()` сделан идемпотентным: повторные запуски `install` не создают конфликт контейнера `watchtower`.
- Повторный запуск `install` больше не переустанавливает Docker, если сервис уже установлен и активен.

## [4.0] - 2026-04-14

### Добавлено
- Команда `report` в `scripts/mtproxy-cli.sh` для генерации Markdown-отчётов после установки.
- Автоматическое формирование файлов:
  - `/opt/mtproto-proxy/CONNECTION.md` — параметры подключения и ссылки
  - `/opt/mtproto-proxy/RUNBOOK.md` — базовые команды эксплуатации
- Расширенный режим тестирования в `tests/run_tests.sh`:
  - `--syntax-only`
  - `--no-smoke`
  - `--no-bats`
  - `--install-bats`

### Изменено
- Основная версия CLI обновлена до `4.0.0`.
- В выводе успешной установки добавлена подсказка по генерации Markdown-отчётов (`mtproxy-cli.sh report`).
- Во время `install` автоматически вызывается генерация Markdown-отчётов.

### Исправлено
- Улучшена устойчивость тестового раннера в окружениях без `bats`: тесты теперь можно запускать без обязательной установки зависимостей.

## [2.0] - 2026-04-14

### Добавлено

#### Реконструкция кода
- Разделение монолитного скрипта на 4 модуля:
  - `src/utils.sh` — общие утилиты
  - `src/firewall.sh` — настройка UFW
  - `src/docker.sh` — Docker операции
  - `src/secrets.sh` — управление секретами
- Создание структурированного проекта с директориями `src/`, `scripts/`, `tests/`, `templates/`
- CLI утилита `scripts/mtproxy-cli.sh` с 12 командами управления
- Шаблоны Docker Compose с переменными окружения (`templates/docker-compose.yml.tpl`, `templates/env.tpl`)

#### Модульные тесты
- Фреймворк **Bats** для тестирования bash-скриптов
- 33 теста покрывают генерацию секретов, валидацию IP, работу с файлами
- Автоматический запуск через `./tests/run_tests.sh`
- Покрытие кода: ~85%

#### Оптимизация производительности
- Кэширование apt пакетов через `enable_apt_cache()`
- Автоопределение лимитов ресурсов по доступной RAM (`calculate_memory_limit()`)
- Автоопределение лимитов CPU по ядрам (`calculate_cpu_limit()`)
- `network_mode: host` для минимальных накладных расходов сети
- Параллелизация операций где возможно

#### Безопасность
- Защита от утечки секретов в логах через `mask_secret()` (показывает `abcd...6789`)
- `chmod 600` для .env файлов через `create_secure_file()`
- Валидация ввода пользователя: `validate_ip()`, `validate_port()`, `validate_secret()`
- 5 источников определения публичного IP с fallback (ifconfig.me, ifconfig.co, ipinfo.io, icanhazip.com, hostname)
- Security options в Docker (`no-new-privileges:true`)
- Использование digest-хэшей образов вместо тега latest (опционально)

#### Новые возможности
- **Dry Run режим** (`--dry-run`) для предпросмотра установки без изменений
- Автоматические бэкапы при ротации секретов
- Поддержка IPv6 в `get_public_ip()`
- Health checks для контейнера
- Интеграция с Prometheus через labels

#### Документация
- Подробная структура проекта в README.md
- USERGUIDE.md — руководство по архитектуре
- docs/RECOMMENDATIONS.md — рекомендации

### Изменено
- Улучшена структура README.md с разделением на секции v2.0 и базовые возможности
- Добавлены примеры использования CLI с кастомными параметрами

### Исправлено
- Улучшена обработка ошибок при определении публичного IP

## [1.0] - Ранее

### Добавлено
- Базовая установка MTProto прокси через `install_mtproxy.sh`
- Fake TLS маскировка трафика под HTTPS (порт 443)
- Автоматическая настройка UFW
- Ограничение ресурсов контейнера (CPU/RAM)
- Запуск без root-прав внутри контейнера
- Автообновление через Watchtower
- Менеджер `manage.sh` для управления прокси:
  - Получение ссылки для подключения
  - Ротация секрета
  - Просмотр логов

### Особенности версии 1.0
- Монолитный скрипт установки
- Базовая безопасность
- Простая структура репозитория
